# Screenshots

This folder contains screenshots for the documentation.

## Required Screenshots

After setting up the project, take screenshots of:

1. **screenshot-dashboard.png** - Main dashboard showing stats and recent releases
2. **screenshot-apps.png** - Apps management page
3. **screenshot-releases.png** - Releases list page
4. **screenshot-release-detail.png** - Single release detail with files

## How to Take Screenshots

1. Start the server: `docker compose up -d`
2. Create some sample data (apps, releases)
3. Take screenshots at 1280x720 or higher
4. Optimize images before committing

You can use tools like:
- Browser DevTools device toolbar
- macOS Screenshot (Cmd+Shift+4)
- ShareX on Windows
